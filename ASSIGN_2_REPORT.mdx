## 1. Introduction

### Project Overview

**Sekuwa Kerner** is a comprehensive e-commerce web application designed for a Nepali food ordering platform. The system specializes in traditional Nepali grilled food items known as "Sekuwa" - a popular street food delicacy in Nepal.

### Purpose

The primary purpose of this project is to create a fully functional online food ordering system that enables:

- **Customers** to browse menu items, add products to cart, place orders, and track order status
- **Administrators** to manage products, process orders, and view business reports

### Technologies Used

| Technology | Purpose |
|------------|---------|
| **PHP 8.x** | Server-side scripting language |
| **MySQL** | Relational database management system |
| **HTML5** | Frontend structure |
| **CSS3** | Styling and responsive design |
| **JavaScript** | Client-side interactivity |
| **XAMPP** | Local development environment |

### Key Features

1. **User Authentication System** - Registration, login, and role-based access control
2. **Product Management** - Full CRUD operations for food items
3. **Shopping Cart** - Session-based cart functionality
4. **Order Processing** - Complete order lifecycle management
5. **Payment Integration** - Support for multiple payment methods (Cash on Delivery, eSewa, Khalti)
6. **Admin Dashboard** - Statistics, order management, and reports
7. **Customer Dashboard** - Order history and tracking

## 2. Database Description

### Database Name: `sekuwa_kerner`

The database is designed following normalization principles to ensure data integrity, minimize redundancy, and maintain efficient relationships between entities.

### Tables Overview

| Table Name | Description | Total Columns |
|------------|-------------|---------------|
| `users` | Stores customer and admin account information | 7 |
| `categories` | Product categories (Chicken Sekuwa, Buff Sekuwa, etc.) | 4 |
| `products` | All food items available for ordering | 9 |
| `orders` | Customer orders with delivery details | 9 |
| `order_items` | Individual items in each order (many-to-many resolution) | 5 |
| `payments` | Payment information for each order | 6 |

### Data Types Used

- **INT** - For primary keys and foreign keys
- **VARCHAR** - For strings with known maximum length
- **TEXT** - For longer text content (descriptions, addresses)
- **DECIMAL(10,2)** - For monetary values (prices)
- **ENUM** - For predefined value sets (status, roles)
- **TIMESTAMP** - For date and time tracking
- **TINYINT(1)** - For boolean flags

### Entity Relationships Explained

1. **Users â†’ Orders (1:M)**
   - One user can place multiple orders
   - Each order belongs to exactly one user
   - `user_id` in orders references `id` in users

2. **Categories â†’ Products (1:M)**
   - One category can have multiple products
   - Each product belongs to one category
   - `category_id` in products references `id` in categories

3. **Orders â†’ Order_Items (1:M)**
   - One order can contain multiple items
   - Each order item belongs to one order
   - `order_id` in order_items references `id` in orders

4. **Products â†’ Order_Items (1:M)**
   - One product can appear in multiple order items
   - Each order item references one product
   - `product_id` in order_items references `id` in products

5. **Orders â†’ Payments (1:1)**
   - Each order has one payment record
   - `order_id` in payments references `id` in orders

### 5.2 Create Tables

#### Users Table

```sql
-- USERS TABLE: Stores customer and admin accounts
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    phone VARCHAR(20),
    password VARCHAR(255) NOT NULL,
    role ENUM('customer', 'admin') DEFAULT 'customer',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;
```

#### Categories Table

```sql
-- CATEGORIES TABLE: Product categories
CREATE TABLE categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;
```

#### Products Table

```sql
-- PRODUCTS TABLE: All food items available for ordering
CREATE TABLE products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    description TEXT,
    category_id INT NOT NULL,
    image VARCHAR(255) DEFAULT 'default.jpg',
    is_featured TINYINT(1) DEFAULT 0,
    is_available TINYINT(1) DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE CASCADE
) ENGINE=InnoDB;
```

#### Orders Table

```sql
-- ORDERS TABLE: Customer orders with delivery details
CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    total_price DECIMAL(10, 2) NOT NULL,
    address TEXT NOT NULL,
    phone VARCHAR(20) NOT NULL,
    status ENUM('Pending', 'Cooking', 'Out for Delivery', 'Delivered', 'Cancelled') DEFAULT 'Pending',
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB;
```

#### Order Items Table

```sql
-- ORDER_ITEMS TABLE: Individual items in each order
CREATE TABLE order_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL DEFAULT 1,
    price DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (order_id) REFERENCES orders(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE
) ENGINE=InnoDB;
```

#### Payments Table

```sql
-- PAYMENTS TABLE: Payment information for each order
CREATE TABLE payments (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT NOT NULL,
    payment_method ENUM('Cash on Delivery', 'eSewa', 'Khalti') NOT NULL,
    payment_status ENUM('Pending', 'Completed', 'Failed') DEFAULT 'Pending',
    transaction_id VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES orders(id) ON DELETE CASCADE
) ENGINE=InnoDB;
```

### 5.4 CRUD Operations

#### CREATE Operation - Adding a New Product

**SQL Statement:**
```sql
INSERT INTO products (name, price, description, category_id, image, is_featured, is_available) 
VALUES (?, ?, ?, ?, ?, ?, ?);
```

**PHP Implementation (admin/add_product.php):**

```php
<?php
// Handle form submission
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Get form data
    $name = sanitize($conn, $_POST['name'] ?? '');
    $description = sanitize($conn, $_POST['description'] ?? '');
    $price = floatval($_POST['price'] ?? 0);
    $category_id = intval($_POST['category_id'] ?? 0);
    $is_featured = isset($_POST['is_featured']) ? 1 : 0;
    $is_available = isset($_POST['is_available']) ? 1 : 0;
    
    // Handle image upload
    $image_name = 'default.jpg';
    if (isset($_FILES['image']) && $_FILES['image']['error'] === UPLOAD_ERR_OK) {
        $allowed_types = ['image/jpeg', 'image/png', 'image/gif', 'image/webp'];
        $file_type = $_FILES['image']['type'];
        
        if (in_array($file_type, $allowed_types)) {
            $ext = pathinfo($_FILES['image']['name'], PATHINFO_EXTENSION);
            $image_name = 'product_' . time() . '_' . rand(1000, 9999) . '.' . $ext;
            $upload_path = '../assets/images/' . $image_name;
            move_uploaded_file($_FILES['image']['tmp_name'], $upload_path);
        }
    }
    
    // Prepared statement to prevent SQL injection
    $stmt = $conn->prepare("INSERT INTO products (name, price, description, category_id, image, is_featured, is_available) 
                            VALUES (?, ?, ?, ?, ?, ?, ?)");
    $stmt->bind_param("sdsissi", $name, $price, $description, $category_id, $image_name, $is_featured, $is_available);
    
    if ($stmt->execute()) {
        setFlashMessage('success', 'Product added successfully!');
        redirect('products.php');
    }
}
?>
```

SELECT p.*, c.name as category_name 
FROM products p 
JOIN categories c ON p.category_id = c.id

        $error = 'Please fill in all fields.';
    } else {
        // Fetch user by email using prepared statement
        $stmt = $conn->prepare("SELECT id, name, email, password, role FROM users WHERE email = ?");
        $stmt->bind_param("s", $email);

        $result = $stmt->get_result();
        
        if ($result->num_rows === 1) {
            $user = $result->fetch_assoc();
            
            // Verify password using password_verify()
            if (password_verify($password, $user['password'])) {
                // Set session variables
                $_SESSION['user_id'] = $user['id'];
                $_SESSION['user_name'] = $user['name'];
                $_SESSION['user_email'] = $user['email'];

                    if (isset($_SESSION['redirect_after_login'])) {
                        $redirect = $_SESSION['redirect_after_login'];
                        unset($_SESSION['redirect_after_login']);
                        redirect($redirect);
                    } else {
                        redirect('../index.php');
                    }
                }
            } else {
                $error = 'Invalid email or password.';
            }
